API para control de vacunas
Descripción: Esta API permite conocer las vacunas que una persona tiene a 
partir de su CURP. Se pueden también registrar nuevas vacunas para una 
persona si se conoce la vacuna que se le administró, la fecha y los datos 
de la persona (CURP).
Parámetros y respuestas esperadas:
ENDPOINT:
http://vacunas.org/api/v1/vacunas/:curp
Verbo: GET
Parámetro de entrada esperado: CURP de la persona.
Salida: JSON con los datos de la persona:
CURP (CURP de la persona buscada)
VACUNAS (lista de las vacunas que se tienen registradas para esa persona, 
los detalles que incluye cada vacuna son: fecha_de_aplicacion, 
nombre_vacuna)
Ejemplo: http://vacunas.org/api/v1/vacunas/GOGL901201HDFNMS04
Devolvería el JSON:
{
"curp": "GOGL901201HDFNMS04",
"vacunas": [
{
"fecha_aplicacion": "14/01/2022 13:34",
"nombre_vacuna": "Pfizer"
},
{
"fecha_aplicacion": "14/02/2022 17:34",
"nombre_vacuna": "Pfizer refuerzo"
}
]
}
ENDPOINT:
Registrar una vacuna para una persona
http://vacunas.org/api/v1/vacunas/registra
Verbo: POST
Parámetro de entrada esperado: CURP de la persona, fecha de aplicación y 
nombre de la vacuna.
Salida: JSON de confirmación con los datos de la persona y de la vacuna:
CURP (CURP de la persona)
fecha_de_aplicacion (La fecha de aplicación recibida)
nombre_vacuna (El nombre de la vacuna recibida)
Ejemplo: http://vacunas.org/api/v1/vacunas/registra (debe ser una petición 
POST con los datos necesarios en el body request)
Devolvería el JSON:
{
"curp": "GOGL901201HDFNMS04",
"fecha_aplicacion": "14/01/2022 13:34",
"nombre_vacuna": "Pfizer"
